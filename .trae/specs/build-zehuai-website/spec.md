# 泽怀影像工作室官网开发 Spec

## Why
泽怀影像工作室目前缺乏线上品牌展示阵地，需要从零构建一个高品质企业官网，用于品牌展示、服务介绍、作品展示和客户转化，让潜在客户能全面了解工作室的新中式电影感写真定位与服务体系。

## What Changes
- 新建完整的 React + TypeScript + Vite + Tailwind CSS 前端项目
- 实现单页应用（SPA），包含 Hero、品牌故事、服务体系、风格展示、服务流程、门店信息、FAQ、关于我们、活动专区、联系预约共 10 个核心区块
- 实现响应式布局，适配桌面端、平板和移动端
- 实现电影感视觉设计风格，匹配品牌调性
- 实现滚动动画和交互效果
- 配置 Vite 开发服务器，确保可直接通过浏览器访问

## Impact
- 新增项目：完整的前端工程化项目
- 技术栈：React 18 + TypeScript + Vite + Tailwind CSS + Framer Motion
- 部署方式：静态站点，可直接部署至任意静态托管服务

---

## ADDED Requirements

### Requirement: 项目初始化与工程化
系统 SHALL 使用 React + TypeScript + Vite + Tailwind CSS 初始化项目，采用 react-ts 模板，项目结构遵循 web-dev-guideline 规范。

#### Scenario: 项目初始化成功
- **WHEN** 执行项目初始化命令
- **THEN** 生成完整的 React + TypeScript + Vite + Tailwind CSS 项目结构，包含 src/components、src/pages、src/hooks、src/utils 等目录

### Requirement: Hero 区块
系统 SHALL 在首屏展示品牌主视觉区域，包含品牌 Slogan、副标题和 CTA 按钮，背景采用电影感视觉风格。

#### Scenario: 用户访问首页
- **WHEN** 用户首次打开网站
- **THEN** 首屏展示全屏 Hero 区，包含品牌 Slogan「光影里有情绪，画面里有故事」、副标题「杭州新中式电影感写真工作室」和「立即预约」CTA 按钮

### Requirement: 品牌故事区块
系统 SHALL 展示品牌故事、品牌定位和核心差异化对比表格，传达新中式电影感写真的独特价值。

#### Scenario: 用户浏览品牌故事
- **WHEN** 用户滚动至品牌故事区块
- **THEN** 展示品牌故事文案、品牌定位说明，以及传统影楼 vs 泽怀影像的三维度对比表（动作/情绪/光影）

### Requirement: 服务体系区块
系统 SHALL 展示单人写真和双人写真套餐的价格表、增值服务和服务承诺。

#### Scenario: 用户查看套餐价格
- **WHEN** 用户浏览服务体系区块
- **THEN** 以卡片或表格形式展示单人三档套餐（基础款1,499元/标准款2,499元/尊享款3,599元）和双人三档套餐，以及增值服务和服务承诺

### Requirement: 风格展示区块
系统 SHALL 展示五种风格细分（新中式旗袍、古风汉服、清新文艺、情侣写真、闺蜜写真），配合外景地和四季主题展示。

#### Scenario: 用户浏览风格展示
- **WHEN** 用户滚动至风格展示区块
- **THEN** 展示五种风格的卡片/画廊，每种风格含关键词和适合人群描述，以及杭州外景地列表和四季主题时间线

### Requirement: 服务流程区块
系统 SHALL 以可视化时间线形式展示从咨询预约到售后保障的完整客户旅程。

#### Scenario: 用户了解服务流程
- **WHEN** 用户浏览服务流程区块
- **THEN** 展示五步流程（咨询预约→拍摄前准备→拍摄当天→后期交付→售后保障），每步含关键信息

### Requirement: 门店信息区块
系统 SHALL 展示 Studio 信息、杭州外景地一览和场地设施说明。

#### Scenario: 用户查看门店信息
- **WHEN** 用户浏览门店信息区块
- **THEN** 展示 Studio 地址、营业时间、联系方式，以及外景地表格和场地设施列表

### Requirement: FAQ 区块
系统 SHALL 以手风琴（Accordion）形式展示常见问题与解答，涵盖价格、拍摄、后期和其他四大类。

#### Scenario: 用户查看常见问题
- **WHEN** 用户点击某个 FAQ 问题
- **THEN** 该问题展开显示答案，其余问题收起

### Requirement: 关于我们区块
系统 SHALL 展示团队介绍、技术实力和数字化管理能力。

#### Scenario: 用户了解团队
- **WHEN** 用户浏览关于我们区块
- **THEN** 展示摄影师/化妆师/客服团队介绍，技术实力表格，以及数字化管理体系说明

### Requirement: 活动专区区块
系统 SHALL 展示常驻活动、季节限定、节日营销和老客户权益。

#### Scenario: 用户查看活动信息
- **WHEN** 用户浏览活动专区
- **THEN** 展示精修满四赠一、闺蜜同行等常驻活动，以及四季限定主题和节日营销信息

### Requirement: 联系预约区块
系统 SHALL 展示微信小程序入口、社交媒体矩阵和联系方式，作为页面底部 CTA。

#### Scenario: 用户想要预约
- **WHEN** 用户浏览至联系预约区块
- **THEN** 展示小程序二维码/入口、小红书/抖音/视频号/朋友圈等社交媒体链接

### Requirement: 响应式布局
系统 SHALL 适配桌面端（≥1280px）、平板（768px-1279px）和移动端（<768px）三种屏幕尺寸。

#### Scenario: 移动端访问
- **WHEN** 用户在手机浏览器打开网站
- **THEN** 页面布局自动调整为单列，导航变为汉堡菜单，字体和间距适配小屏

### Requirement: 导航系统
系统 SHALL 提供固定顶部导航栏，包含各区块锚点链接，支持平滑滚动跳转。

#### Scenario: 用户点击导航链接
- **WHEN** 用户点击导航栏中的「服务体系」
- **THEN** 页面平滑滚动至服务体系区块

### Requirement: 滚动动画
系统 SHALL 在各区块进入视口时触发入场动画，包括淡入、上移等效果。

#### Scenario: 区块进入视口
- **WHEN** 用户滚动页面使某区块进入可视区域
- **THEN** 该区块内容以淡入上移动画方式出现

### Requirement: 视觉设计风格
系统 SHALL 采用电影感视觉设计，主色调为深色系（墨黑/深灰），辅以暖金色点缀，字体选用具有东方韵味的衬线体搭配现代无衬线体，整体氛围沉稳、文艺、有质感。

#### Scenario: 视觉一致性
- **WHEN** 用户浏览整个网站
- **THEN** 所有区块保持统一的色彩体系、字体规范和间距节奏，呈现电影感新中式美学
